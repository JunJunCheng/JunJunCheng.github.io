    • Why Virtual Destructor

    Virtual destructors are useful when you can delete an instance of a derived class through a pointer to base class:

    And the output is:

    delete pa 实际上相当于：pa->~Animal();
    释放pa所指向的内存。在这里，因为~Animal是virtual的，尽管是通过Animal类型的指针调用的，根据v-table的信息，~Dog() 被正确调用。如果把virtual属性去掉，那么被调用的是~Animal()，Dog类的connstructor被调用而deconstructor未被调用，constructor中分配的资源没有释放，从而产生了内存泄漏

    If set the base class destructor not virtual, the output will change to:

    The destructor of class Dog is not called. In most implementations, the call to the destructor will be resolved like any non-virtual code, meaning that the destructor of the base class will be called but not the one of the derived class, resulting in a resources leak.

    To sum up, always make base classes’ destructors virtual when they’re meant to be manipulated polymorphically.
